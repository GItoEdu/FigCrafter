# Undo機能の修正とUI・機能の拡張計画

## 目的
ユーザーからのフィードバックに基づき、Undo機能の欠落箇所を修正し、UIの操作性とファイル取り込み時の利便性を向上させます。

## ユーザー確認事項
- **消しゴムのUndo**: 消しゴム操作（ストローク全体）が1つの操作として元に戻せるようになります。
- **不透明度設定**: 数値入力に加え、直感的に操作できるスライダーを追加します。
- **インポート挙動**: 画像や他のファイルを読み込んだ際、自動的に新しいレイヤーが作成され、そこに配置されるようになります。

## 提案される変更点

### 1. Undo/Redo 全般の改善
- **ショートカット有効化**: `MainWindow.xaml` に `Ctrl+Z`, `Ctrl+Y` の `KeyBinding` を追加します。
- **色変更の履歴登録**: `MainViewModel.cs` の色変更処理を `ExecuteCommand` 経由に修正します。
- **重要プロパティの挙動修正**: `FillColor`, `StrokeColor`, `StrokeWidth` を変更した際、即座にメニューの「元に戻す」が活性化するようにします。

### 2. 消しゴム機能のUndo対応
- **EraserCommandの新設**: 消しゴム操作によるマスク（SKBitmap）の変更前後の状態を保持するコマンドを実装します。
- **CanvasView.xaml.csの調整**: マウス操作の開始（Down）と終了（Up）で、マスク状態をコピー・比較してコマンドを発行するようにします。

### 3. UIの改善（不透明度スライダー）
- **MainWindow.xaml**: 不透明度の `TextBox` の横に、0.0〜1.0 の範囲で動作する `Slider` を配置します。

### 4. インポート機能の強化
- **MainViewModel.cs**: `OpenFileAsync`（画像インポート時）において、画像を追加する前に自動的に `AddLayer()` を呼び出すように変更します。

## 検証方法
- [ ] 消しゴムで画像を消した後、`Ctrl+Z` で元に戻るか。
- [ ] スライダーで不透明度を調整し、即座にキャンバスに反映されるか。
- [ ] 画像ファイルをドラッグ＆ドロップまたは「開く」からインポートした際、新しいレイヤーが作成されているか。
- [ ] `Ctrl+Z` でインポート（レイヤー作成＋画像追加）が一度に元に戻るか（必要に応じてCompositeCommandを使用）。
