# FigCrafter - Undo機能修正とインポート/UI改善

このディレクトリでは、**FigCrafter** におけるUndo機能（`Ctrl+Z` / `Ctrl+Y`）のバグ修正や全般的な対象範囲拡大、ならびに画像インポート時の利便性向上、UIの改良に関する実装記録を保管しています。

## 概要

以前のバージョンでは、色変更や消しゴム操作、インポートといった一部の操作がUndoの履歴（スタック）に乗らず、またショートカットキーも一部で効かない問題がありました。本アップデートにより、これらの操作をすべてコマンドパターン（`IUndoableCommand`）に統合し、一貫したUndo/Redo体験を提供できるようになりました。同時に、ユーザビリティの観点から不透明度スライダーの追加など小規模な拡張も行っています。

## 主な変更点

1. **全体的なUndo/Redoの安定化**
    - `MainWindow.xaml` に `Ctrl+Z` と `Ctrl+Y` のキーバインディングを追加し、フォーカス状態によらずUndo/Redoを実行できるようにしました。
    - キャンバス上の「塗り色設定（FillColor）」や「線の色設定（StrokeColor）」のUI操作をコマンド化し、右パネルからの操作でも履歴に残るように修正しました (`CanvasViewModel.cs`, `MainViewModel.cs`)。
    - Undoコマンドが実行された際に `RaiseCanExecuteChanged` を呼び出し、UIヘッダー側のUndoボタンも即座に有効化されるように同期を改善しました。
2. **消しゴム機能のUndo対応と枠線の調整**
    - `EraserCommand.cs` を新設し、消しゴムストローク開始時と終了時のマスク用ビットマップ（`SKBitmap`）を保持、前後状態を復元できるようにしました。
    - 消しゴムでドラッグ中の領域を示す破線の枠線を細く（StrokeWidth=0.5）し、より描画対象が見えやすくなるように調整しました。
3. **インポート画像の自動レイヤー化と一括Undo**
    - 「ファイルを開く」やドラッグ&ドロップによる画像インポート時、既存のレイヤーを汚さず、自動的に「新規レイヤー」を作成して配置するように変更しました。
    - レイヤー追加と画像追加を `CompositeCommand` として一つの履歴にまとめたことで、インポート後に `Ctrl+Z` を1回押すだけで「追加されたレイヤーと画像」の両方がまとめて消える（元に戻る）ように設計しました (`AddLayerCommand.cs` の新設)。
4. **不透明度スライダーの実装**
    - プロパティパネル（`MainWindow.xaml`）の「不透明度」項目に `Slider` コントロールを追加しました（0.0～1.0の範囲）。数値の入力をせずとも直感的に図形の透明度を調整できます。
