# UIレイアウト改良と画像DPI/座標系の修正

本タスクでは、FigCrafterのUIレイアウトの抜本的な見直しと、インポートした画像の表示サイズが正しくない（DPIおよび単位系の不一致）問題の根本的な解決を行いました。

## 変更内容

### 1. UIの視認性向上とレイアウト再構成
- **メニューバーの新設**: ファイル、編集、表示の標準的なメニューバーを上部に配置しました。
- **プロパティツールバー**: 選択中のオブジェクトのプロパティ（X, Y, W, H, 回転, 不透明度）や整列・グループ化等の操作をアイコン付きで横並びに配置しました。
- **図形ツールバー**: キャンバス直上のツールバーとして独立・アイコン化させ、アクセス性を向上させました。
- **右パネルの整理**: レイヤーパネルを最上部に移動し、カラーパレット（37色）やRGB（HEX）入力などの機能を追加しました。
- **視認性の改善**: 背景に埋もれていたメニュー文字の修正や、ツールバーボタンのサイズ拡大（クリックしやすさの向上）を行いました。

### 2. TIFFファイルインポート対応
- SkiaSharpで非対応だったTIFF形式を、`System.Drawing.Bitmap` を経由して読み込むように `ImportHelper.cs` を修正しました。これにより、TIFFファイルを安全にインポートできるようになりました。

### 3. 画像DPIとキャンバス座標系の根本的修正
- **問題の原因**: オブジェクトのプロパティ値（X, Y, Width, Height）がミリメートル（mm）として扱われている一方、キャンバスの描画がWPFの論理ピクセル（96DPI基準）で行われていたため、表示サイズが本来の約1/4に縮小される不具合がありました。
- **キャンバス座標系のmm単位統一**:
  - `CanvasView` の描画処理（`SKPaintSurface`）の基点に `96.0 / 25.4` のスケール変換を導入しました。これにより、全ての `GraphicObject` をミリメートル単位のまま描画できるようになりました。
  - マウス座標変換にも逆方向の係数を適用し、ミリメートル座標系での正確な操作（移動・リサイズ・スナップ等）を可能にしました。
- **結果**: 82.7mm幅の画像などが、210mm幅のアアートボードに対して正しい物理的な比率で表示・操作されるようになり、プロパティ数値と見た目の整合性が完全に取れました。

## 検証結果

### 自動テスト
- `dotnet build` を実行し、ビルドが正常に完了することを確認しました。

### 手動確認
1. **インポートサイズ**: 82.7mm幅の画像をインポートし、210mm幅のキャンバス上で正しく約4割の幅を占めることを目視確認（ユーザー様より「表示も正しくなった」との確認済み）。
2. **操作性**: マウスでのオブジェクトドラッグ、リサイズ、スナップ機能が正しく動作することを確認。
3. **UI表示**: メニューバー、ステータスバー、各ボタンのサイズや視認性が良好であることを確認。
