# UIレイアウト改良

現状の右サイドバーにすべてのUI要素が詰め込まれ、スクロール頻度が高い問題を解決する。
メニューバー、プロパティツールバー、図形ツールバーを分離し、右パネルをレイヤー＋色設定に特化させる。

## 現状の問題

- 右パネル（250px幅）にアートボード設定、描画ツール、オブジェクトプロパティ（X/Y/W/H/回転/不透明度）、色選択（4色のみ）、整列、グループ、削除、レイヤーパネルがすべて詰め込まれている
- 色は固定7色（SkyBlue, Salmon, LightGreen, Black, Red, Blue, Transparent）のみ
- RGB入力なし

## 提案変更

### MainWindow.xaml

#### [MODIFY] [MainWindow.xaml](i:/Ubuntu/VSCode/FigCrafter/MainWindow.xaml)

**1. メインGrid Row定義の変更**
- Row0: Auto → メニューバー (`Menu`)
- Row1: Auto → プロパティツールバー
- Row2: Auto → 図形ツールバー
- Row3: * → メインコンテンツ (TabControl + 右パネル)
- Row4: Auto → ステータスバー

**2. Row 0: メニューバー (Menu) 追加**
- ファイル(F): 新規(Ctrl+N) / 開く(Ctrl+O) / 上書き保存(Ctrl+S) / 別名で保存(Ctrl+Shift+S) / セパレータ / PNG抽出 / PDF抽出 / TIF抽出
- 編集(E): 元に戻す(Ctrl+Z) / やり直し(Ctrl+Y) / セパレータ / 切り取り(Ctrl+X) / コピー(Ctrl+C) / 貼り付け(Ctrl+V) / 全選択(Ctrl+A)
- 表示(V): スナップ切り替え(Ctrl+U)

**3. Row 1: プロパティツールバー (オブジェクト選択時表示)**
- 横一列にコンパクト配置:
  - X / Y / W / H / 回転 / 不透明度の入力フィールド
  - 整列ボタン6個（アイコン化）
  - 重ね順ボタン4個（アイコン化）
  - グループ化/解除ボタン
  - 削除ボタン

**4. Row 2: 図形ツールバー (アイコン横並び)**
- 現在の2列RadioButtonを、アイコン付き横並びに変更
- ツール: 選択 / 四角形 / 円 / 直線 / テキスト / 消しゴム

**5. 右パネル再構成 (順序変更)**
1. **レイヤーパネル** → 最上部に移動
2. **アートボード設定** → キャンバス名/幅/高さ（折りたたみ可能）
3. **線設定** → 線の太さ + 枠線色パレット
4. **塗り色設定** → 塗り色パレット + RGB入力
5. **型固有設定** → テキスト/画像/直線の専用UI

### カラーパレット仕様

**塗り色・枠線色共通で以下のカラーパレットを提供:**

| カテゴリ | 色数 | 内容 |
|---------|------|------|
| 基本色 | 12 | 黒, 白, 赤, 橙, 黄, 緑, 水, 青, 紫, ピンク, 茶, 灰 |
| 明るい色 | 12 | 各基本色の明るいバリエーション |
| 暗い色 | 12 | 各基本色の暗いバリエーション |
| 透明 | 1 | 透明（✕マーク表示） |

合計37色 + RGB直接入力

**RGB入力UI:**
- `#` + 6桁HEXテキストボックス
- 現在色のプレビュー表示
